---
title: Install
description: Install @smoove/renderer to export a composition to an MP4 file from Node.
---

`@smoove/renderer` turns a composition into a video file. It runs in Node,
not the browser: it rasterizes each frame with skia-canvas and encodes with
[Mediabunny](https://mediabunny.dev), all in-process. Install it alongside
`@smoove/core`.

```bash
npm install konva @smoove/core @smoove/renderer
# or
pnpm add konva @smoove/core @smoove/renderer
```

<Callout type="warn" title="Native requirements">
This package is Node-only. `skia-canvas` builds a native module, so the install
machine needs a system C/C++ toolchain. Media decoding and encoding come from
Mediabunny (`@mediabunny/server`, which uses prebuilt FFmpeg bindings), pulled in
as dependencies, so there is no separate ffmpeg binary to install. None of this
runs in the browser; use the [Player](/docs/player) there instead.
</Callout>

Set up the server rendering backend once, before you build the composition:

```ts
import "@smoove/renderer/register"; // calls setupServerRendering() on import
import { Composition } from "@smoove/core";
import { renderComposition } from "@smoove/renderer";
```

<Callout type="info" title="Shader transitions need the WebGL path">
Shader (Tier B) presentations from `@smoove/transitions` render through the
renderer's WebGL path. Install transitions and the optional `gl` dependency, then
import the `gl` entry before building the composition:

```bash
npm install @smoove/transitions
npm install --save-optional gl
```

```ts
import "@smoove/renderer/gl";
```

`@smoove/transitions` is an optional peer; without `gl`, shader transitions
fall back to a plain `fade()`.
</Callout>
