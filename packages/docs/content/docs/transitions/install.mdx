---
title: Install
description: Install @smoove/transitions to blend scenes with cross-fades, slides, wipes, and shader effects.
---

`TransitionSeries` and the presentations (`fade`, `slide`, `wipe`, and the rest)
ship in their own package, `@smoove/transitions`. It builds on
`@smoove/core`, so add it once you have a composition and want to blend one
scene into the next.

```bash
npm install konva @smoove/core @smoove/transitions
# or
pnpm add konva @smoove/core @smoove/transitions
```

<Callout type="info" title="konva and core are peers">
You install `konva` and `@smoove/core` yourself; the transitions package
lists them as peer dependencies. Pin a single copy of `konva` so `instanceof`
checks and layout keep working. See [Installation](/docs/installation) for the
core setup.
</Callout>

A series needs a `Composition` up front (spring timings read its `fps`, and
presentations are sized from its `width`/`height`):

```ts
import { Composition } from "@smoove/core";
import { TransitionSeries, fade, linearTiming } from "@smoove/transitions";

const sceneLen = 80;
const blend = 20;
const comp = new Composition({
  id: "intro",
  fps: 60,
  durationInFrames: 2 * sceneLen - blend, // net duration: scenes minus the shared overlap
  width: 1280,
  height: 720,
});
const series = new TransitionSeries({ composition: comp });
```

<Callout type="info" title="Shader presentations render headless with the renderer">
The shader presentations (`dissolve`, `crosswarp`, `bookFlip`, and the rest) run
on WebGL2 in the browser. To render them in a headless export, the renderer needs
its WebGL path. See [Rendering setup](/docs/rendering/install).
</Callout>
