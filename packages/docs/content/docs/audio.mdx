title: Audio
description: Add sound to a composition, automate levels from the frame, and balance every track through one master mixer.

`Audio` adds a sound track. It is an invisible node you add to a
[Sequence](/docs/sequencing): it draws nothing, but the sequence's
`from`/`durationInFrames` range-gate when it plays, and the composition's mixer
sounds it on the frame clock. Levels are driven by the frame, the same as
everything else, so a mix scrubs and renders deterministically.

## Install

`Audio` ships in the `@smoove/media` package, which owns the mediabunny
decoder so `@smoove/core` stays free of it:

```bash
npm install @smoove/media
# or
pnpm add @smoove/media
```

Importing the package registers its browser decoders, so in the browser
there is nothing else to wire up.

```ts
import { Composition, Sequence } from "@smoove/core";
import { Audio } from "@smoove/media";

const fps = 60;
const comp = new Composition({ id: "scene", fps, durationInFrames: fps * 26, width: 1080, height: 1080 });

const music = new Audio({ src: musicUrl, name: "Music", volume: 0.8 });
const seq = new Sequence();
seq.add(music);
comp.add(seq);
```

## Props

`src` is the only required prop.

| Prop | Values |
| --- | --- |
| `src` | URL string of the audio file (required) |
| `id` | stable id for the track (useful for render and mixer UIs) |
| `name` | human label shown in mixer UIs (defaults to `src`) |
| `trimBefore` | first frame of the clip to play (skips the head) |
| `trimAfter` | last frame to play (cuts the tail) |
| `loop` | `boolean`: restart the clip when it reaches the end |
| `muted` | `boolean`: silence this track |
| `volume` | number `0..1`: this track's level before the master mix |
| `playbackRate` | number: `1` is real time, `2` is double speed |

A track is heard only while the playhead is inside its sequence's window. Put a
sound effect on a short `Sequence({ from, durationInFrames })` and it fires for
exactly that range, then stops.

## Automate volume from the frame

Drive a track's level the recommended way: from the frame, with
[`interpolate`](/docs/interpolation) inside a `register` updater. Fade a music
bed in over the first half second, hold it, then fade it out at the end.

```ts
import { interpolate } from "@smoove/core";

const FADE = 30; // half a second at 60fps
seq.register((frame) => {
  const level = interpolate(
    frame,
    [0, FADE, 1530, 1560],
    [0, 0.8, 0.8, 0],
    { extrapolateLeft: "clamp", extrapolateRight: "clamp" },
  );
  music.setVolume(level);
});
```

<Callout type="warn" title="Set volume with setVolume(), not by assignment">
`audio.volume` is a read-only signal you read with `audio.volume.get()`, so you
cannot assign to it. Change the level with `audio.setVolume(0..1)`, mute with
`audio.setMuted(true)`, and change speed with `audio.setPlaybackRate(n)`.
</Callout>

## Read the sound

Volume automation is the level you set. Sometimes a visual needs the level of
the sound itself: a VU meter, a waveform, motion that lands on the beat. Pass
`introspect: true` and the clip's loudness becomes a frame-pure signal.

```ts
const music = new Audio({ src: musicUrl, introspect: true });

seq.register((f) => {
  meter.width(300 * music.rmsAt(f));                      // real loudness, 0..1
  holdTick.x(300 * music.peakAt(f, { holdFrames: 18 }));  // decaying peak hold
});
```

`rmsAt(f)` and `peakAt(f)` take the same local frame your updater receives and
respect `trim`, `loop`, and `playbackRate`. They read a small envelope decoded
once before frame 0, so scrubbing and server renders see identical values. The
readings are pre-fader (the file's own loudness): multiply by your gain for a
post-fader meter, or pass `{ normalized: true }` to scale against the clip's
own loudest moment. Both return 0 until the envelope is ready, and `introspect`
costs one extra decode of the clip, so enable it only where a visual reads it.

For a spectrum, ask for bands:

```ts
const music = new Audio({ src: musicUrl, introspect: { bands: 24 } });

seq.register((f) => {
  const bands = music.bandsAt(f);          // 24 log-spaced magnitudes, bass → treble
  bands.forEach((v, k) => eqBars[k].height(200 * v));
  if (music.noveltyAt(f) > 0.7) kick();    // onset spike → beat-synced motion
});
```

The same machinery works outside a composition too: `buildEnvelope` decodes
any file into an `AudioEnvelope`, and the `envelopeRmsAt` family reads it by
media time in seconds. No `Audio` node needed.

```ts
import { buildEnvelope, envelopeRmsAt } from "@smoove/media";

const env = await buildEnvelope("/audio/track.mp3", { bands: 24 });
envelopeRmsAt(env, 12.5); // loudness at 12.5s into the file
```

If you know what you are doing, you can drop all the way to the decoder:
`audio.source` returns the live media source, and on the default
`MediabunnyAudioSource` (and its video counterpart) `source.input` is the
underlying mediabunny `Input`, ready for track metadata or extra sinks. Read
from it; the source owns its lifecycle, so never dispose it yourself.

`waveform(fromFrame, toFrame, buckets)` returns the clip's static min/max
outline: one signed pair per column, ready to draw as bars or a polygon. The
outline never changes with playback, so build it once when the envelope lands
(it returns all zeros while the decode is still in flight):

```ts
const podcast = new Audio({ src: voiceUrl, introspect: true });

let drawn = false;
seq.register(() => {
  if (drawn) return;
  const wf = podcast.waveform(0, DURATION, 120); // 120 columns
  if (wf.max.every((v) => v === 0)) return;      // envelope still decoding
  drawn = true;
  wf.max.forEach((hi, i) => {
    const lo = wf.min[i] ?? 0;
    bars[i].y(MID - hi * SCALE);                 // top edge
    bars[i].height((hi - lo) * SCALE);           // down to the bottom edge
  });
});
```

The raw table is on `audio.envelope` if you want to build something else from
it.

## The master mixer

Every composition has a master bus at `comp.mixer`. It scales and gates every
track at once. A track's effective level is `master × track`, and it is silent
when either the master or the track is muted.

```ts
comp.mixer.setVolume(0.5); // halve everything
comp.mixer.setMuted(true); // silence the whole mix

comp.mixer.volume.get(); // read the master level
comp.mixer.channels;      // every registered Video/Audio track, for a meter UI
```

Layer as many tracks as you want: a music bed, a voiceover on its own range, a
couple of one-shot effects. Mixing is just frame-driven volume curves per track.
This demo crossfades two music beds, ducks the music under a voice with an
`interpolate` range, and fires whooshes on short sequences, all on one playhead.

<Callout type="info" title="Renders headless too">
On a headless render the mixer does not play audio, it records each track as it
sounds (level, mute, position). `@smoove/renderer` then decodes, mixes, and
encodes every track in-process with [Mediabunny](https://mediabunny.dev), no
ffmpeg. The same frame-driven volume curves produce the rendered mix. Add the
package from [Rendering setup](/docs/rendering/install), then see
[Rendering media](/docs/rendering/media).
</Callout>

## Probe metadata

`probeMedia(src)` reads a track's container metadata without decoding it, so
a beat can last exactly as long as its voice-over:

```ts
import { probeMedia } from "@smoove/media";

const meta = await probeMedia("/audio/vo.mp3");
meta.duration; // seconds
meta.sampleRate;
meta.channels;
meta.durationInFrames(fps); // floor(duration * fps)
```

Await it at module top level, before building the comp. See
[Timeline markers](/docs/markers#plan-the-timeline-first) for the planning
workflow this feeds.
