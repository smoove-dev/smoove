smoove ships as an ES module with `konva` as a peer dependency. The fastest
way to a running project is the scaffolding CLI; if you'd rather wire it into
an existing app, add the package by hand.

<Tabs items={["npm create smoove", "Manual install"]}>

<Tab value="npm create smoove">

## Scaffold a project

```bash
npm create smoove
# or, non-interactively
npm create smoove composition my-video -- --ts
```

You'll be prompted for a template and a target directory. Two templates:

- **composition**: a minimal Vite app, one composition, `<smoove-player>`
  preview, autoreload. TypeScript or JavaScript.
- **studio**: the full studio, browse compositions, scrub the timeline,
  tweak props, and render MP4s on a built-in server queue.

The CLI installs dependencies, runs `git init`, and offers to add the
`smoove-video` agent skill so a coding agent can write compositions for you.
See [Agent skills](/docs/skills) for what that installs and why.

## Run it

```bash
cd my-video
npm run dev
```

Open the printed URL. The player previews `src/composition.ts`; edit it and
the preview reloads.

<Callout type="tip" title="Options">
`--ts` / `--js` (composition template only), `--no-install`, `--no-git`,
`--no-skill`. Anything you don't pass becomes an interactive prompt.
</Callout>

## Install

```bash
npm install konva @smoove/core
# or
pnpm add konva @smoove/core
```

<Callout type="info" title="Konva is a peer dependency">
smoove **extends** Konva's classes. `Composition` is a `Konva.Stage`,
`Sequence` is a `Konva.Layer`, and `Rect`/`Circle`/`Text`/… extend their Konva
equivalents. Install `konva` yourself and make sure your bundler resolves a
single copy; two copies break `instanceof` checks and layout.
</Callout>

## Your first composition

Three pieces: a `Composition` (the frame clock), a `Sequence` (a range-gated
layer), and an **updater** that sets attributes as a function of the frame.

```ts
import { Circle, Composition, Easing, Rect, Sequence, interpolate } from "@smoove/core";

const fps = 60;
const comp = new Composition({
  id: "first-tween",
  fps,
  durationInFrames: fps * 3,
  width: 1280,
  height: 720,
  loop: true,
});

const scene = new Sequence();
scene.add(new Rect({ x: 0, y: 0, width: 1280, height: 720, fill: "#0d1117" }));

const ball = new Circle({ x: 0, y: 360, radius: 64, fill: "#4ea1ff" });
scene.add(ball);

scene.register((frame) => {
  // "Where is the ball at frame N?" The whole model in one line.
  ball.x(interpolate(frame, [0, 179], [160, 1120], { easing: Easing.inOut(Easing.cubic) }));
  ball.opacity(interpolate(frame, [0, 30, 150, 179], [0, 1, 1, 0]));
});

comp.add(scene);

// Mount into a DOM element and play.
comp.setContainer(document.getElementById("scene") as HTMLDivElement);
comp.play();
```

<Callout type="warn" title="Add the node before you animate it">
Attach every node to a `Sequence` (`scene.add(node)`) **before** you tween it.
Updating attributes on a detached node changes the values but nothing repaints.
The sequence is what draws it.

## Step by step

Same three pieces either way you got here: a `Composition` (the frame clock),
a `Sequence` (a range-gated layer), and an **updater** that sets attributes as
a function of the frame.

<Steps>

<Step>
### Create a composition

`new Composition({ id, fps, durationInFrames, width, height, loop })` defines the
frame clock and the canvas. `fps × durationInFrames` is the whole timeline.
</Step>

<Step>
### Add a sequence

`new Sequence({ from, durationInFrames })` is a range-gated layer. Both options
default: `from` to `0` and `durationInFrames` to the composition's duration, so
`new Sequence()` spans the whole timeline. Add your nodes to it with
`seq.add(...)`, then add the sequence to the composition with `comp.add(seq)`.
</Step>

<Step>
### Register a frame updater

`seq.register((frame) => { … })` runs every tick the sequence is in range. Set
attributes from the frame, usually via [`interpolate`](/docs/interpolation) and
[`Easing`](/docs/easing). The argument is the **sequence-local** frame.
</Step>

<Step>
### Mount and play

Point the stage at a container with `comp.setContainer(el)` (or pass `container`
in the constructor), then `comp.play()`. For offline or server rendering use
`comp.setFrame(n)` instead. It works without a browser.
</Step>

</Steps>

To feed data into a composition (a headline, a name, a color), pass it as
`props` and read it back in your updaters. That's how one composition renders
many different videos; see [Dynamic props](/docs/dynamic-props).

Next: [Core concepts](/docs/core-concepts) goes deeper on range-gating and the
tick loop, or jump to [Animating](/docs/interpolation) to learn `interpolate`
and `Easing`. [Best practices](/docs/best-practices) covers the conventions
this example already follows, like sizing `durationInFrames` from `fps`.
